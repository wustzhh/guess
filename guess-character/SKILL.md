# 猜人物游戏

## 游戏规则

1. **Agent 选角色**：从用户指定的范围（如三国、水浒、金庸等）中暗选一个人物，不透露任何信息
2. **用户提问**：用户通过提问来缩小范围
3. **Agent 回答限制**：只能回答以下三种：
   - **是** — 肯定
   - **不是** — 否定
   - **是也不是** — 附带一句简短解释（不超过10字）
4. **禁止主动暗示**：在用户猜出之前，绝不能给出明显暗示
5. **猜中后科普**：用户猜对后，输出该人物的生平简介
6. **战绩汇总**：科普结束后，输出本轮战绩表
7. **战绩持久化**：每局结束（猜中或放弃）后，将记录存入数据库并生成 HTML 报告

## 游戏流程

### 第一步：确认范围

用户指定范围后，从中选一个**有一定知名度但不是最热门**的角色（增加趣味性）。

回复：`好，我已经想好了一个{范围}的人物。开始提问吧！`

**内部记录**：Agent 需在内部记录以下信息，供结算时使用：
- 选定的角色名
- 所属分类（如"三国"、"水浒"等）
- 每轮的提问和回答

### 第二步：回答提问

严格遵守回答限制，保持简洁。

常见提问维度参考（不要主动提示这些维度）：
- 时间线（生卒年、活跃时期）
- 阵营/门派/国家
- 身份类型（文/武/谋士/君主）
- 关键事件参与
- 人物关系

### 第三步：猜中后科普 / 放弃后揭晓

**猜中**：用加粗标题输出人物信息，包含：
- 姓名、字号、生卒年
- 出身背景
- 主要事迹（按时间线，3-5条）
- 最终结局
- 一句话历史评价

**放弃**：公布答案，同样输出上述科普内容。

### 第四步：战绩汇总

输出 Markdown 表格：

```
## 猜人战绩单

| 轮次 | 你的提问 | 我的回答 |
|------|---------|---------|
| 1    | xxx？   | 是/不是 |
| ...  | ...     | ...     |
| N    | **xxx！** | **正确！** / **放弃** |

**总计：X次提问 + 1次猜测 = Y轮猜出答案**
或
**总计：X次提问后放弃**

评价：{根据轮次给出趣味评价}
```

趣味评价参考：
- 1-5轮：神机妙算！
- 6-10轮：思路清晰，效率很高！
- 11-15轮：稳扎稳打，不错！
- 16轮以上：锲而不舍，终于猜到了！
- 放弃：虽败犹荣，下次一定！

### 第五步：保存战绩并生成报告（必须执行）

游戏结束后（无论猜中还是放弃），**必须**执行以下命令保存战绩：

```bash
python D:\pyProj\guess\game_db.py save "<分类>" "<角色名>" <提问数> <总轮次> <结果> '<问答日志JSON>'
```

参数说明：
- `<分类>`：如 "三国"、"水浒"、"金庸" 等
- `<角色名>`：本局答案
- `<提问数>`：用户提出的问题数量（不含最终猜测/放弃）
- `<总轮次>`：所有交互轮次（包括猜测/放弃）
- `<结果>`：`guessed`（猜中）或 `gave_up`（放弃）
- `<问答日志JSON>`：JSON 数组，格式 `[{"q":"问题","a":"回答"}, ...]`

示例：
```bash
python D:\pyProj\guess\game_db.py save "三国" "陆逊" 7 8 guessed '[{"q":"死于234年之前还是之后？","a":"之后"},{"q":"是蜀国的吗？","a":"不是"}]'
```

脚本会自动保存到 SQLite 数据库并生成 HTML 战绩报告页面。

保存成功后告知用户：**战绩已保存！打开 `D:\pyProj\guess\stats.html` 可查看历史战绩。**

## 注意事项

- 选角色时避免过于冷门（用户可能完全不知道）
- 如果用户猜错，只回复"不是"，不要给额外提示
- 如果用户提问不是是非题，引导用户改为是非题形式
- 用户可以随时放弃，放弃时公布答案并科普
- **每局结束必须保存战绩**，这是游戏流程的必要步骤
