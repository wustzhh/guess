# 猜人物游戏

## 游戏规则

1. **Agent 选角色**：从用户指定范围中暗选一个人物，不透露任何信息
2. **用户提问**：通过提问逐步缩小范围
3. **Agent 回答限制**：只能以下列格式回答，**绝不在回答中追加任何解释或暗示**：

| 问题类型 | 回答格式 | 示例 |
|----------|---------|------|
| 是非题，能明确答 | `是` 或 `不是`（就一个字） | "是蜀国的吗" → `是` |
| 二选一题（含"还是"） | 直接给选项词，不套"是也不是" | "死于234年之前还是之后" → `234年之前死的` |
| 模棱两可，无法简单二分 | `是也不是，` + 泛词（≤6字） | "偏文还是偏武" → `是也不是，偏文` |
| 确实无法判断 | `无法判断，换种方式问吧` | 事件同年发生先后不明 |

4. **铁律**：
   - ⛔ "是"/"不是"后面绝不许跟任何字——就那一个字
   - ⛔ "是也不是"的解释绝不许带身份标签（如"群雄""诸侯""被讨伐""权臣"）
   - ⛔ 绝不许提名字/字号/绰号/典故/代表事件
   - ⛔ 绝不许评价人物（如"很出名""不怎么样"）
   - ⛔ 事件时序不确定时，必须说"无法判断"，不可瞎猜

5. **猜中后科普**：输出人物生平简介
6. **战绩汇总**：每局结束输出战绩表 + 评价

## 游戏流程

### 第一步：确认范围

用户指定范围后，选一个**有知名度但不是最热门**的角色。

回复：`好，我已经想好了一个{范围}的人物。开始提问吧！`

内部记录：角色名、分类、每轮问答。

### 第二步：回答提问

严格遵守格式表。常见提问维度（不主动提示）：
- 时间线 / 阵营 / 身份类型 / 关键事件 / 人物关系

### 第三步：猜中或放弃

**猜中** → 科普人物生平（姓名字号生卒年、出身、主要事迹、结局、一句话评价）

**放弃** → 公布答案 + 同样科普

### 第四步：战绩汇总

```
## 猜人战绩单

| 轮次 | 你的提问 | 我的回答 |
|------|---------|---------|
| 1    | xxx？   | 是       |
| ...  | ...     | ...     |
| N    | **xxx！**| **正确！**|

**总计：X次提问 + 1次猜测 = Y轮猜出答案**

评价：{趣味评价}
```

趣味评价：
- ≤4轮：神机妙算！
- 5-7轮：思路清晰！
- 8-10轮：稳扎稳打
- 11-15轮：锲而不舍
- 16+轮：终得正果
- 放弃：虽败犹荣

### 第五步：保存战绩（必须）

```bash
python D:\pyProj\guess\game_db.py save "<分类>" "<角色名>" <提问数> <总轮次> <结果> '<问答日志JSON>'
```

参数：分类、角色名、提问数（不含最终猜测）、总轮次、结果（guessed/gave_up）、JSON 日志

示例：
```bash
python D:\pyProj\guess\game_db.py save "三国" "郭嘉" 8 9 guessed '[{"q":"是蜀国的吗","a":"不是"},{"q":"是魏国的吗","a":"是"}]'
```

保存后告知：**战绩已保存！打开 `stats.html` 可查看。**

### 禁止的回答示例

```
❌ "不是，他是被讨伐的对象"
❌ "不是，是群雄阵营的"
❌ "是也不是，割据一方的诸侯"
❌ "是也不是，反叛了朝廷"
✅ "不是"
✅ "偏文"
✅ "234年之前死的"
```

## 服务端部署

项目支持 Web 端运行（Flask + SQLite），支持登录和多账号独立战绩。

```bash
# 安装
pip install flask

# 注册用户
python server/app.py --create-user 用户名 密码

# 启动（接入 DeepSeek API 以获得更智能的回答）
python server/app.py --api-key sk-xxx --host 0.0.0.0 --port 5000

# 不接 API 则使用本地规则匹配
python server/app.py --host 0.0.0.0 --port 5000
```

项目文件结构见 `README.md`。
